Projects
01 — Web App
Resume Builder
A live, in-browser resume editor with PDF export, color themes, AI-generated summaries, and a clean two-panel layout. Built using Claude and ChatGPT — no frameworks, just HTML, CSS, and vanilla JS.
View Live ↗
Here's how the project actually came together — the prompts I used, why I phrased them that way, and what I was trying to get out of each exchange.
My Prompt Claude
Build me a resume builder that works entirely in the browser. Two-panel layout — editable form on the left, live preview on the right. The preview should update in real time as I type. No backend, no frameworks. Include: personal info, work experience, education, and skills. Make the design clean and modern.
What I got
A working two-panel layout with form inputs wired to a live preview via JavaScript event listeners. Clean typography, structured sections. Functional from the first output — I didn't have to explain the layout twice.
Why this worked: I gave it the constraints upfront ("no backend, no frameworks") so it didn't over-engineer it, and I led with the interaction pattern ("live preview as you type") so the JS structure was right from the start. Specific inputs → fewer revision loops.
My Prompt Claude
Add accent color themes — at least 6 options the user can pick from. The color should apply to the resume preview: headers, dividers, name styling. Keep the selector clean, nothing flashy. Also add auto-save to localStorage so nothing is lost on refresh.
What I got
Color picker with 8 named themes (Amber, Cobalt, Forest, etc.) using CSS custom properties. localStorage save/load wired to every input change. The preview updated instantly on theme switch.
Why this worked: I bundled two related features in one prompt because they both touch the same data layer. Asking for them separately would've created two rounds of code that might conflict. Knowing when to combine requests matters as much as knowing how to phrase them.
My Prompt ChatGPT
I want an "AI Write" button next to the summary field. When clicked, it should call the Anthropic API using the user's entered name, title, and work experience, and return a 2–3 sentence professional summary. Show a loading state while it runs. Keep the button small and inline.
What I got
Async fetch call to the Anthropic messages endpoint, pulling live values from the form fields to build the prompt dynamically. Loading spinner inline with the button. Populated the textarea on success.
Why this worked: I told it exactly where the data was coming from ("the user's entered name, title, and work experience") so it knew to read from the DOM, not hardcode anything. I also specified the UX detail I cared about ("show a loading state") so I didn't have to come back for it.
My Prompt Claude
Add a PDF download using the browser's print API — no third-party libraries. Style the print view so only the resume preview renders, full-width, with no UI chrome. The accent color should carry through to the printed version.
What I got
@media print CSS that hides all controls and expands the preview to full width. window.print() wired to the download button. The CSS custom property for accent color was already in scope so it rendered correctly in the print dialog.
Why this worked: Specifying "no third-party libraries" was the key constraint — it forced an elegant native solution rather than adding a dependency. The reminder about accent color wasn't obvious; it only works because the CSS variable architecture was already set up from the themes prompt.
